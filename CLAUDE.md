# ぱくぱくフレンズ 開発メモ(Claude Code用)

4歳児向けスマホ知育ゲーム。依存なし・ビルドなし(ES modules直配信)。
実際に4歳児が遊んでおり、その反応をもとに反復改善している。

## ファイル構成

- `index.html` — HTML + CSS(見た目はすべてここ)
- `js/main.js` — ゲーム本体(状態・画面遷移・各お世話あそび・ごほうび)
- `js/data.js` — 定数データ(おともだち・食べ物・色・形・帽子・褒め言葉・鳴き声データ)
- `js/audio.js` — WebAudio効果音(`tone`/`sfx`/`cry`)と英語読み上げ(`speakEn`)

## デプロイ

- リポジトリ: https://github.com/t11086/pakupaku-friends
- 公開URL: **https://t11086.github.io/pakupaku-friends/** (GitHub Pages, mainブランチ直配信)
- 手順: `git add` → `git commit` → `git push` で自動デプロイ(反映まで1〜3分)
- **注意: Pagesのビルドがたまに詰まる。** 5分待っても反映されない場合は手動リビルド:
  `gh api repos/t11086/pakupaku-friends/pages/builds -X POST`
  状態確認: `gh api repos/t11086/pakupaku-friends/pages/builds/latest --jq '.status + " " + .commit[0:7]'`

## 変更後の検証手順

1. JSの構文チェック: `node --check js/main.js js/data.js js/audio.js`(1ファイルずつ)
2. ローカル確認: `python3 -m http.server 8787`(このディレクトリで。ES modulesなので file:// では動かない)
3. デプロイ後、`curl` で新しい文字列が配信されているか確認(キャッシュ回避に `?v=$(date +%s)` を付ける)。JSを変えた場合は `js/main.js` 等も直接curlすること

## アーキテクチャ方針(ユーザー確認済み 2026-07-06)

- **React化はしない。** ビルドなし・単一HTMLの身軽さ(push→1分で反映)がこのプロジェクトの強み
- ただし index.html の見通しが悪くなってきたら、**ES modules での数ファイル分割(ビルドなしのまま)を Claude が自主的に提案・実施してよい**(ユーザー承諾済み)

## 設計ルール(ユーザー確認済み・変更しないこと)

- **日本語の合成音声(speechSynthesis ja-JP)は使わない**。不自然なため無効化済み(`speak()` はno-op)
- **英語音声のみ使用**: `speakEn()` — 明るい高めの声(rate .9 / pitch 1.4)。褒め言葉・食べ物・色・動物の名前を読み上げ。英語学習を兼ねる
- 文字が読めない前提: ボタンは絵文字メイン、文字だけのUIは不可(例: シャワーボタンは🚿のみ+脈打つアニメ)
- シャワーボタンの背景は**白**。青は🚿の水色と同化し、オレンジはハレーションを起こすとNGが出た(2026-07)
- 要素の表示/非表示で高さが変わるとキャラがカクッと動く。消すときは textContent='' ではなく visibility を使う(ねんねの🌞で発生した)
- 失敗・減点・時間制限なし。タスク完了時は必ず褒める(画面は👍絵文字のみ+英語音声。テキスト褒め言葉は改行が不自然になるため廃止)
- 何度遊んでも毎回完了の反応を出す(達成済みフラグで completeTask をスキップしない — 過去に「2回目無反応」バグの原因になった)
- スマホ縦画面前提: 高さは `100dvh`(URLバー対策)、要素はコンパクトに。PWA対応済み(manifest.json + アイコン)
- タップ対象は大きく(おもちゃ16vw等)。小さいと4歳児が押せない

## ゲーム構成(現状)

- お世話5種: 🍚ごはん / 🪥はみがき / 🛁おふろ / 🧸おかたづけ / 🌙ねんね → スタンプ5個で1日クリア
- クリア後のごほうびは4日ローテーション: 🎈ふうせんわり → 🎨いろあわせ → ⭐かたちあわせ → 🔢かずあわせ(各3問、間違えても揺れて再出題のみでペナルティなし)。かずあわせのボタンは数字+同じ数の●で数字が読めなくてもOK
- ずかん12匹コンプリートの日は特別演出: 🏆メッセージ+全員パレード+紙吹雪増量。以降の全クリア日もパレード表示
- ごはんはぱっくんに3つ食べさせた後、おうえんおともだちがいれば1つおすそわけしてから完了("Yummy! Thank you!")
- 📖ずかん: クリアごとに動物のおともだち+1(全12匹、localStorage保存)。タップで英語名
- ホームに集めたおともだちが遊びに来る(最大4匹、タップ反応あり)
- おうえんおともだち: **ごはん画面のみ**に登場(ひとくちごとに応援ジャンプ+おすそわけ)。**他のお世話画面には出さない** — 一度全画面に出したが「意味を感じない」とユーザー判断で削除(2026-07)
- ホームのぱっくんはタップで反応: ぴょんと跳ねて "Hello!" / "Let's play!" 等のあいさつ
- ごはんの英語カウント: 食べるたび "One! Two! Three!"(`speakEn(text, queue)` の queue=true で名前の読み上げを消さず続けて話す)
- きせかえ帽子: 日数で自動アップグレード(3日🎀 / 5日🧢 / 7日🎩 / 10日👑 / 13日🎓 / 16日🌟 / 20日🌈)。全キャラに表示。サイズは container query(cqw)基準
- 帽子が変わる日は、次の日に進むとき🎁プレゼント開封のお披露目演出が入る
- おともだちは動物ごとに鳴き声効果音(WebAudioのチップチューン風、`friendCries`)+英語名。ホーム・おうえん・ずかん共通
- 英語単語: 食べ物・色・形・動物名・おもちゃ名・カウントに加え、はみがき "Clean teeth!" / おふろ "Bubbles!" "Splash, splash!" / あさ "Good morning!"
- いろあわせ・かたちあわせは出題時と間違えたときに "Which is red?" / "Which is the star?" と英語で問いかける
