# ぱくぱくフレンズ 開発メモ(Claude Code用)

4歳児向けスマホ知育ゲーム。単一の `index.html` で完結(依存なし・ビルドなし)。
実際に4歳児が遊んでおり、その反応をもとに反復改善している。

## デプロイ

- リポジトリ: https://github.com/t11086/pakupaku-friends
- 公開URL: **https://t11086.github.io/pakupaku-friends/** (GitHub Pages, mainブランチ直配信)
- 手順: `git add` → `git commit` → `git push` で自動デプロイ(反映まで1〜3分)
- **注意: Pagesのビルドがたまに詰まる。** 5分待っても反映されない場合は手動リビルド:
  `gh api repos/t11086/pakupaku-friends/pages/builds -X POST`
  状態確認: `gh api repos/t11086/pakupaku-friends/pages/builds/latest --jq '.status + " " + .commit[0:7]'`

## 変更後の検証手順

1. インラインJSの構文チェック: `<script>` 部を抽出して `node --check`
2. ローカル確認: `python3 -m http.server 8787`(このディレクトリで)
3. デプロイ後、`curl` で新しい文字列が配信されているか確認(キャッシュ回避に `?v=$(date +%s)` を付ける)

## 設計ルール(ユーザー確認済み・変更しないこと)

- **日本語の合成音声(speechSynthesis ja-JP)は使わない**。不自然なため無効化済み(`speak()` はno-op)
- **英語音声のみ使用**: `speakEn()` — 明るい高めの声(rate .9 / pitch 1.4)。褒め言葉・食べ物・色・動物の名前を読み上げ。英語学習を兼ねる
- 文字が読めない前提: ボタンは絵文字メイン、文字だけのUIは不可(例: シャワーボタンは🚿のみ+脈打つアニメ)
- 失敗・減点・時間制限なし。必ず「できた!💮」で褒める
- 何度遊んでも毎回完了の反応を出す(達成済みフラグで completeTask をスキップしない — 過去に「2回目無反応」バグの原因になった)
- スマホ縦画面前提: 高さは `100dvh`(URLバー対策)、要素はコンパクトに。PWA対応済み(manifest.json + アイコン)
- タップ対象は大きく(おもちゃ16vw等)。小さいと4歳児が押せない

## ゲーム構成(現状)

- お世話5種: 🍚ごはん / 🪥はみがき / 🛁おふろ / 🧸おかたづけ / 🌙ねんね → スタンプ5個で1日クリア
- クリア後: 🎈ふうせんわり(ごほうび、色の英語)→ 次の日へ
- 📖ずかん: クリアごとに動物のおともだち+1(全12匹、localStorage保存)。タップで英語名
- ホームに集めたおともだちが遊びに来る(最大4匹、タップ反応あり)

## 次のアップデート候補(ユーザー未承認のアイデア)

- おともだちと一緒にお世話する演出
- 日数に応じたぱっくんの成長・きせかえ(帽子などのアクセサリー)
- ごはんの英語カウント("One! Two! Three!")
